let const

let注意：
1.没有域解析，不存在变量提升
在代码快内，只要let定义变量,在之前使用，都是报错
先定义，在使用
2.同一个作用域里，不能重复定义变量
3.for循环，for循环里面是父级作用域，里面又一个块级作用域

const注意：
const特性和let一样
const定义完变量，必须有值，不能后赋值，不能修改
Object,freeze(对象)锁定这个对象，不能进行增加减少

es5增加的
1.forEach第一个参数是回调函数，第二个参数是this.的指向 。没有return

2.map 非常有用，做数据交互映射
        正常情况下，需要配合return，返回一个新的数组
        若没有return相当于forEach
        注意：平时只要用map，一定要有return
        它可以重新整理数据
        [{title:'aaa'}]->[{t:'aaa'}]
3.arr.filter() 过滤 过滤一些不合格'元素'，如果回调函数返回true，就留下来
4.arr.some() 类似查找，数组里面某一个元素合格，返回true
5.arr.every() 数组里面的元素都要符合，才返回true
上面他们的参数都一样（val,index,arr）
6.arr,reduce((上一个，当前这个，index，arr)) //从左往右 ，可以求数组的和
7arr.redrceRight()//从右往左

模块化

import 特点
a) import 可以是相对路径 ，也可以是绝对路劲
b) import 模块只会导入一次，无论你引多少次
c) import ‘./views/1.js’, 如果这么用，相当于引入文件
d)有提升效果，import会自动提升到顶部,首行执行
e) 导出去的模块类容，如果里面有定时器更改，外面也会改动，不想Common规范存

import()类似node里面的require,可以动态引入，默认Import语法不能写在if之类里面
返回值，是个promise对象
import(‘./views/1.js’).thien(res=>{
console.log(res.a+res.b)
})
